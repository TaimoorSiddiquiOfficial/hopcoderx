HopCoderX CLI は、引数なしで実行すると、デフォルトで [TUI](/docs/tui) を開始します。

```bash
HopCoderX
```

ただし、このページに記載されているようにコマンドも受け入れます。これにより、HopCoderX をプログラム的に操作できるようになります。

```bash
HopCoderX run "Explain how closures work in JavaScript"
```

---

### tui

HopCoderX ターミナルユーザーインターフェイスを開始します。

```bash
HopCoderX [project]
```

#### フラグ

| フラグ       | ショート    | 説明                                                       |
| ------------ | ----------- | ---------------------------------------------------------- |
| `--continue` | `-c`        | 最後のセッションを続行                                     |
| `--session`  |             | 続行時にセッションをフォーク (`-s` または `--fork` と併用) |
| `--continue` | `--session` | 続行するセッション ID                                      |
| `--prompt`   |             | 使用のプロンプト                                           |
| `--model`    | `-m`        | プロバイダー/モデルの形式で使用するモデル                  |
| `--agent`    |             | 使用するエージェント                                       |
| `--port`     |             | リッスンするポート                                         |
| `--hostname` |             | リッスンするホスト名                                       |

## コマンド

HopCoderX CLI には次のコマンドもあります。

---

### agent

HopCoderX のエージェントを管理します。

```bash
HopCoderX agent [command]
```

---

### attach

`serve` または `web` コマンドを使用して起動された、すでに実行中の HopCoderX バックエンドサーバーにターミナルを接続します。

```bash
HopCoderX attach [url]
```

これにより、リモート HopCoderX バックエンドで TUI を使用できるようになります。例えば：

```bash
# Start the backend server for web/mobile access
HopCoderX web --port 4096 --hostname 0.0.0.0

# In another terminal, attach the TUI to the running backend
HopCoderX attach http://10.20.30.40:4096
```

#### フラグ

| フラグ      | ショート | 説明                           |
| ----------- | -------- | ------------------------------ |
| `--dir`     |          | TUI を開始する作業ディレクトリ |
| `--session` | `-s`     | 続行するセッション ID          |

---

#### create

カスタム構成で新しいエージェントを作成します。

```bash
HopCoderX agent create
```

このコマンドは、カスタムシステムプロンプトとツール構成を使用して新しいエージェントを作成する手順を示します。

---

#### list

利用可能なエージェントをすべてリストします。

```bash
HopCoderX agent list
```

---

### auth

プロバイダーの資格情報とログインを管理するコマンド。

```bash
HopCoderX auth [command]
```

---

#### login

HopCoderX は [Models.dev](https://models.dev) のプロバイダーリストを利用しているため、`HopCoderX auth login` を使用して、使用したいプロバイダーの API キーを構成できます。これは `~/.local/share/HopCoderX/auth.json` に保存されます。

```bash
HopCoderX auth login
```

HopCoderX が起動すると、認証情報ファイルからプロバイダーがロードされます。また、環境またはプロジェクト内の `.env` ファイルで定義されたキーがあるかどうかも確認します。

---

#### list

認証情報ファイルに保存されているすべての認証されたプロバイダーをリストします。

```bash
HopCoderX auth list
```

またはショートバージョン。

```bash
HopCoderX auth ls
```

---

#### logout

資格情報ファイルからプロバイダーをクリアすることで、プロバイダーからログアウトします。

```bash
HopCoderX auth logout
```

### github

リポジトリ自動化のための GitHub エージェントを管理します。

```bash
HopCoderX github [command]
```

---

#### install

GitHub エージェントをリポジトリにインストールします。

```bash
HopCoderX github install
```

これにより、必要な GitHub Actions ワークフローが設定され、構成プロセスがガイドされます。 [詳細はこちら](/docs/github)。

---

#### run

GitHub エージェントを実行します。これは通常、GitHub Actions で使用されます。

```bash
HopCoderX github run
```

##### フラグ

| フラグ    | 説明                                                |
| --------- | --------------------------------------------------- |
| `--event` | エージェントを実行するための GitHub モック イベント |
| `--token` | GitHub 個人アクセストークン                         |

### mcp

モデルコンテキストプロトコルサーバーを管理します。

```bash
HopCoderX mcp [command]
```

---

#### add

MCP サーバーを構成に追加します。

```bash
HopCoderX mcp add
```

このコマンドは、ローカルまたはリモートの MCP サーバーを追加する手順を示します。

---

#### list

構成されているすべての MCP サーバーとその接続ステータスをリストします。

```bash
HopCoderX mcp list
```

または、短いバージョンを使用してください。

```bash
HopCoderX mcp ls
```

---

#### auth

OAuth 対応の MCP サーバーで認証します。

```bash
HopCoderX mcp auth [name]
```

サーバー名を指定しない場合は、利用可能な OAuth 対応サーバーから選択するように求められます。

OAuth 対応サーバーとその認証ステータスを一覧表示することもできます。

```bash
HopCoderX mcp auth list
```

または、短いバージョンを使用してください。

```bash
HopCoderX mcp auth ls
```

---

#### logout

MCP サーバーの OAuth 資格情報を削除します。

```bash
HopCoderX mcp logout [name]
```

---

#### debug

MCP サーバーの OAuth 接続の問題をデバッグします。

```bash
HopCoderX mcp debug <name>
```

---

### models

構成されたプロバイダーから利用可能なすべてのモデルをリストします。

```bash
HopCoderX models [provider]
```

このコマンドは、構成されたプロバイダー全体で利用可能なすべてのモデルを `provider/model` の形式で表示します。

これは、[設定](/docs/config/) で使用する正確なモデル名を把握するのに役立ちます。

オプションでプロバイダー ID を渡して、そのプロバイダーによってモデルをフィルターできます。

```bash
HopCoderX models anthropic
```

#### フラグ

| フラグ      | 説明                                                            |
| ----------- | --------------------------------------------------------------- |
| `--refresh` | models.dev からモデルキャッシュを更新します。                   |
| `--verbose` | より詳細なモデル出力を使用します (コストなどのメタデータを含む) |

`--refresh` フラグを使用して、キャッシュされたモデルリストを更新します。これは、新しいモデルがプロバイダーに追加され、それを HopCoderX で確認したい場合に便利です。

```bash
HopCoderX models --refresh
```

### run

プロンプトを直接渡して、非対話モードで HopCoderX を実行します。

```bash
HopCoderX run [message..]
```

これは、スクリプト作成、自動化、または完全な TUI を起動せずに迅速な回答が必要な場合に便利です。例えば。

```bash "HopCoderX run"
HopCoderX run Explain the use of context in Go
```

実行中の `HopCoderX serve` インスタンスにアタッチして、実行ごとの MCP サーバーのコールドブート時間を回避することもできます。

```bash
# Start a headless server in one terminal
HopCoderX serve

# In another terminal, run commands that attach to it
HopCoderX run --attach http://localhost:4096 "Explain async/await in JavaScript"
```

#### フラグ

| フラグ       | ショート    | 説明                                                                                      |
| ------------ | ----------- | ----------------------------------------------------------------------------------------- |
| `--command`  |             | 実行するコマンド。引数には message を使用します。                                         |
| `--continue` | `-c`        | 最後のセッションを続行                                                                    |
| `--session`  |             | 続行時にセッションをフォーク (`-s` または `--fork` と併用)                                |
| `--continue` | `--session` | 続行するセッション ID                                                                     |
| `--share`    |             | セッションを共有する                                                                      |
| `--model`    | `-m`        | プロバイダー/モデルの形式で使用するモデル                                                 |
| `--agent`    |             | 使用するエージェント                                                                      |
| `--file`     | `-f`        | メッセージに添付するファイル                                                              |
| `--format`   |             | 形式: デフォルト (フォーマット済み) または json (生の JSON イベント)                      |
| `--title`    |             | セッションのタイトル (値が指定されていない場合は、切り詰められたプロンプトが使用されます) |
| `--attach`   |             | 実行中の HopCoderX サーバー (http://localhost:4096 など) に接続します。                    |
| `--port`     |             | ローカルサーバーのポート (デフォルトはランダムポート)                                     |

### serve

API アクセスのためにヘッドレス HopCoderX サーバーを起動します。完全な HTTP インターフェイスについては、[サーバーのドキュメント](/docs/server) を確認してください。

```bash
HopCoderX serve
```

これにより、TUI インターフェイスを使用せずに HopCoderX 機能への API アクセスを提供する HTTP サーバーが起動します。 `HOPCODERX_SERVER_PASSWORD` を設定して HTTP 基本認証を有効にします (ユーザー名のデフォルトは `HopCoderX`)。

#### フラグ

| フラグ       | 説明                                    |
| ------------ | --------------------------------------- |
| `--port`     | リッスンするポート                      |
| `--hostname` | リッスンするホスト名                    |
| `--mdns`     | mDNS 検出を有効にする                   |
| `--cors`     | CORS を許可する追加のブラウザーオリジン |

### session

HopCoderX セッションを管理します。

```bash
HopCoderX session [command]
```

---

#### list

すべての HopCoderX セッションをリストします。

```bash
HopCoderX session list
```

##### フラグ

| フラグ        | ショート | 説明                                     |
| ------------- | -------- | ---------------------------------------- |
| `--max-count` | `-n`     | 最新のセッションを N 個に制限            |
| `--format`    |          | 出力形式: テーブルまたは json (テーブル) |

---

### stats

HopCoderX セッションのトークンの使用状況とコストの統計を表示します。

```bash
HopCoderX stats
```

#### フラグ

| フラグ      | 説明                                                                                       |
| ----------- | ------------------------------------------------------------------------------------------ |
| `--days`    | 過去 N 日間の統計を表示 (全期間)                                                           |
| `--tools`   | 表示するツールの数 (すべて)                                                                |
| `--models`  | モデルの使用状況の内訳を表示 (デフォルトでは非表示)。上位 N 件を表示するには数値を渡します |
| `--project` | プロジェクトでフィルタリング (全プロジェクト、空文字列: 現在のプロジェクト)                |

---

### export

セッションデータを JSON としてエクスポートします。

```bash
HopCoderX export [sessionID]
```

セッション ID を指定しない場合は、利用可能なセッションから選択するように求められます。

---

### import

JSON ファイルまたは HopCoderX 共有 URL からセッションデータをインポートします。

```bash
HopCoderX import <file>
```

ローカルファイルまたは HopCoderX 共有 URL からインポートできます。

```bash
HopCoderX import session.json
HopCoderX import https://opncd.ai/s/abc123
```

---

### web

Web インターフェイスを使用してヘッドレス HopCoderX サーバーを起動します。

```bash
HopCoderX web
```

これにより、HTTP サーバーが起動し、Web ブラウザが開き、Web インターフェイスを通じて HopCoderX にアクセスします。 `HOPCODERX_SERVER_PASSWORD` を設定して HTTP 基本認証を有効にします (ユーザー名のデフォルトは `HopCoderX`)。

#### フラグ

| フラグ       | 説明                                    |
| ------------ | --------------------------------------- |
| `--port`     | リッスンするポート                      |
| `--hostname` | リッスンするホスト名                    |
| `--mdns`     | mDNS 検出を有効にする                   |
| `--cors`     | CORS を許可する追加のブラウザーオリジン |

---

### acp

ACP (エージェントクライアントプロトコル) サーバーを起動します。

```bash
HopCoderX acp
```

このコマンドは、nd-JSON を使用して stdin/stdout 経由で通信する ACP サーバーを起動します。

#### フラグ

| フラグ       | 説明                 |
| ------------ | -------------------- |
| `--cwd`      | 作業ディレクトリ     |
| `--port`     | リッスンするポート   |
| `--hostname` | リッスンするホスト名 |

---

### uninstall

HopCoderX をアンインストールし、関連ファイルをすべて削除します。

```bash
HopCoderX uninstall
```

#### フラグ

| フラグ          | ショート | 説明                                         |
| --------------- | -------- | -------------------------------------------- |
| `--keep-config` | `-c`     | 構成ファイルを保持する                       |
| `--keep-data`   | `-d`     | セッションデータとスナップショットを保持する |
| `--dry-run`     |          | 削除せずに削除される内容を表示する           |
| `--force`       | `-f`     | 確認プロンプトをスキップする                 |

---

### upgrade

HopCoderX を最新バージョンまたは特定のバージョンに更新します。

```bash
HopCoderX upgrade [target]
```

最新バージョンにアップグレードするには。

```bash
HopCoderX upgrade
```

特定のバージョンにアップグレードするには。

```bash
HopCoderX upgrade v0.1.48
```

#### フラグ

| フラグ     | ショート | 説明                                                   |
| ---------- | -------- | ------------------------------------------------------ |
| `--method` | `-m`     | 使用されたインストール方法。curl, npm, pnpm, bun, brew |

## 環境変数

HopCoderX は環境変数を使用して構成できます。

| 変数                                  | タイプ   | 説明                                                                |
| ------------------------------------- | -------- | ------------------------------------------------------------------- |
| `HOPCODERX_AUTO_SHARE`                 | ブール値 | セッションを自動的に共有する                                        |
| `HOPCODERX_GIT_BASH_PATH`              | 文字列   | Windows 上で実行可能な Git Bash へのパス                            |
| `HOPCODERX_CONFIG`                     | 文字列   | 構成ファイルへのパス                                                |
| `HOPCODERX_CONFIG_DIR`                 | 文字列   | config ディレクトリへのパス                                         |
| `HOPCODERX_CONFIG_CONTENT`             | 文字列   | インライン JSON 構成コンテンツ                                      |
| `HOPCODERX_DISABLE_AUTOUPDATE`         | ブール値 | 自動更新チェックを無効にする                                        |
| `HOPCODERX_DISABLE_PRUNE`              | ブール値 | 古いデータのプルーニングを無効にする                                |
| `HOPCODERX_DISABLE_TERMINAL_TITLE`     | ブール値 | ターミナルタイトルの自動更新を無効にする                            |
| `HOPCODERX_PERMISSION`                 | 文字列   | インライン化された json 権限設定                                    |
| `HOPCODERX_DISABLE_DEFAULT_PLUGINS`    | ブール値 | デフォルトのプラグインを無効にする                                  |
| `HOPCODERX_DISABLE_LSP_DOWNLOAD`       | ブール値 | LSP サーバーの自動ダウンロードを無効にする                          |
| `HOPCODERX_ENABLE_EXPERIMENTAL_MODELS` | ブール値 | 実験モデルを有効にする                                              |
| `HOPCODERX_DISABLE_AUTOCOMPACT`        | ブール値 | 自動コンテキスト圧縮を無効にする                                    |
| `HOPCODERX_DISABLE_CLAUDE_CODE`        | ブール値 | `.claude` からの読み取りを無効にする (プロンプト + スキル)          |
| `HOPCODERX_DISABLE_CLAUDE_CODE_PROMPT` | ブール値 | `~/.claude/CLAUDE.md` の読み取りを無効にする                        |
| `HOPCODERX_DISABLE_CLAUDE_CODE_SKILLS` | ブール値 | `.claude/skills` のロードを無効にする                               |
| `HOPCODERX_DISABLE_MODELS_FETCH`       | ブール値 | リモートソースからのモデルの取得を無効にする                        |
| `HOPCODERX_FAKE_VCS`                   | 文字列   | テスト目的の偽の VCS プロバイダー                                   |
| `HOPCODERX_DISABLE_FILETIME_CHECK`     | ブール値 | 最適化のためにファイル時間チェックを無効にする                      |
| `HOPCODERX_CLIENT`                     | 文字列   | クライアント識別子 (デフォルトは `cli`)                             |
| `HOPCODERX_ENABLE_EXA`                 | ブール値 | Exa Web 検索ツールを有効にする                                      |
| `HOPCODERX_SERVER_PASSWORD`            | 文字列   | `serve`/`web` の基本認証を有効にする                                |
| `HOPCODERX_SERVER_USERNAME`            | 文字列   | 基本認証ユーザー名 (デフォルト `HopCoderX`) をオーバーライドします。 |
| `HOPCODERX_MODELS_URL`                 | 文字列   | モデル設定を取得するためのカスタム URL                              |
